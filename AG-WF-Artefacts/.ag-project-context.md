# IEG Trading Platform — Project Context

## 1. Technology Stack

- **Languages:** C# / .NET 10.0 (net10.0)
- **Frameworks/Libraries:**
  - ASP.NET Core Web API (controller-based)
  - gRPC (Grpc.AspNetCore 2.67.0, Grpc.Net.Client 2.67.0, Google.Protobuf 3.29.3)
  - Polly 7.2.3 (resilience / retry policies)
  - Consul 1.6.10.9 (service discovery)
  - FluentFTP 54.1.0 (FTP file access)
  - Swashbuckle / Swagger (OpenAPI documentation)
- **Databases/Storage:** In-memory static lists (decentralized data management per service). No SQL/NoSQL database.
- **Testing:** None configured.
- **Tooling/Infrastructure:** dotnet CLI, Swagger UI, HashiCorp Consul, shell scripts (`start-all.sh` / `start-all.bat`).

## 2. Architecture Overview

- **System Type:** Microservices (7 independent ASP.NET Core projects in one solution `SolTradingPlatform.sln`).
- **Layering & Separation:** Each service follows a Controllers → Services (Repository pattern) → In-memory datastore layering. Models are in dedicated `Models/` folders.
- **Data Flow & Interfaces:**
  - **MeiShop** (API Gateway, ports 7024/5009): Orchestrates calls to ProductService, FtpProductCatalogService, and IEGEasyCreditcardService via HTTP. Uses `ResilientServiceCaller` with Polly retry + round-robin load balancing.
  - **ProductService** (port 7200): GET `/api/Products` — returns product list from in-memory store.
  - **FtpProductCatalogService** (port 7300): GET `/api/ProductCatalog` — loads product catalog via FTP.
  - **IEGEasyCreditcardService** (ports 7231/7232/7233): Credit card validation, transactions. 3 instances for horizontal scaling. Uses Consul for service discovery.
  - **PaymentService** (port 7400): CRUD payments with Content Negotiation (JSON/XML/CSV).
  - **LoggingService** (port 5500): Centralized gRPC-based error logging, writes to `logs/error_log.txt`.
  - **Consul** (port 8500): Service Discovery & Configuration.

## 3. Key Discoveries & Patterns

- **Coding Standards:** Nullable enabled, implicit usings enabled. German comments throughout. Controller routing: `[Route("api/[controller]")]` with `[ApiController]`.
- **Design Patterns:**
  - Repository pattern with interface + concrete class (e.g., `IProductRepository` → `ProductRepository`).
  - Dependency injection via constructor (scoped repositories, singleton for load balancer & gRPC client).
  - Round-robin load balancing (`RoundRobinLoadBalancer`) as singleton.
  - Resilient HTTP calling with Polly retry + exponential backoff + failover across instances.
  - Content Negotiation in PaymentService via custom `CsvOutputFormatter`/`CsvInputFormatter` + XML serializer formatters.
  - gRPC for internal logging communication (proto-first approach with `.proto` files in `Protos/` folders).
  - Consul-based service discovery in CashDeskController (legacy pattern).
- **Quirks & Debt:**
  - `CashDeskController` in `IEGEasyCreditcardService` uses `BlackFriday.Controllers` namespace (legacy, mismatched namespace).
  - Some controllers instantiate `new HttpClient()` directly instead of using `IHttpClientFactory` (e.g., ProductListController, ProductCatalogController).
  - Static in-memory data stores (no persistence).
  - All projects target `net10.0`.
  - No unit tests exist.
  - Self-signed certificates accepted for development (`DangerousAcceptAnyServerCertificateValidator`).
